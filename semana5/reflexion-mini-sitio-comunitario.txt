Escribe una reflexión de 8 a 10 líneas respondiendo:

¿Por qué un mini-sitio que consume una API puede ayudar a una comunidad a difundir información de forma responsable, 
organizada y verificable?

En tu respuesta menciona al menos cinco de estas ideas:

- proyecto único en GitHub
- rama main
- rama personal
- git pull origin main
- fast-forward
- async
- await
- fetch
- API
- JSON
- difusión responsable
- planificación del calendario editorial
- calendario editorial
- revisión de fuentes
- consistencia comunicativa
- moderación editorial básica
- lenguaje responsable y no estigmatizante
- participación ciudadana
- cultura de paz

Respuesta:
Un mini-sitio que consume una API ayuda a la comunidad porque permite organizar y mostrar información de forma clara, 
verificable y actualizada usando JSON, fetch, async y await; además, trabajar en un proyecto único en GitHub 
facilita la colaboración manteniendo una rama main estable y una rama personal para cada integrante, mientras 
herramientas como git pull origin main y fast-forward ayudan a mantener el proyecto ordenado; también, 
el calendario editorial, la revisión de fuentes y la moderación editorial básica permiten una difusión 
responsable con consistencia comunicativa, promoviendo participación ciudadana, cultura de paz y lenguaje 
responsable y no estigmatizante.
